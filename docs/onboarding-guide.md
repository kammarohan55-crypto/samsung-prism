# Acme Corp Onboarding Guide

## Welcome to Acme Corp! 🎉

This guide walks you through everything you need to complete in your first 30 days.

## Week 1: Getting Started

### Day 1 — Orientation
1. Meet with HR for paperwork and badge setup
2. Receive your company laptop and peripherals
3. Attend the New Hire Orientation session (1 hour)
4. Complete mandatory compliance training modules

### Day 2 — IT Setup
1. Set up your email (Gmail) and calendar
2. Install VPN client and verify remote access
3. Set up Slack and join required channels:
   - #general, #engineering, #onboarding, #random
4. Set up development environment (see IT Setup Guide)

### Day 3–5 — Team Integration
1. Schedule 1:1 with your direct manager
2. Meet your onboarding buddy
3. Review the team wiki and project documentation
4. Attend your first team standup

## Week 2: Learning the Ropes

### Engineering Onboarding
1. Clone the main repositories from GitHub
2. Follow the local development setup guide
3. Complete your first "good first issue" ticket
4. Get your first Pull Request reviewed and merged

### Culture & Process
1. Read the Engineering Principles document
2. Understand the sprint cycle (2-week sprints)
3. Learn the deployment process and CI/CD pipeline
4. Review incident response procedures

## Week 3–4: Building Momentum

### Goals
1. Complete at least 2 feature tickets independently
2. Participate in a code review for another engineer
3. Present a 5-minute "New Hire Share" at the all-hands meeting
4. Complete your 30-day self-assessment form

### Resources
- **Engineering Wiki**: wiki.acme.com/engineering
- **HR Portal**: hr.acme.com
- **IT Help Desk**: #it-support on Slack or helpdesk@acme.com
- **Your Onboarding Buddy**: Listed in your welcome email

## Completion Criteria
You're considered "fully onboarded" when:
- [ ] All Week 1 tasks completed
- [ ] Dev environment set up and first PR merged
- [ ] All mandatory training completed
- [ ] 30-day self-assessment submitted
- [ ] Manager confirms onboarding completion
